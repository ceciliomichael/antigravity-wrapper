# Refactor Edit Key Flow to Modal

## Overview
Replace the cramped inline table editing with a clean, centered Modal dialog for API Key editing. This resolves overflow issues and provides a better user experience for managing complex data like model permissions.

## Architecture

### 1. Reusable Modal Component
Create `frontend/src/components/ui/modal.tsx` using the existing `Portal` component.
- **Features**: Backdrop with blur, centered content, click-outside to close, escape key support.
- **Props**: `isOpen`, `onClose`, `title`, `children`.

### 2. Edit Key Feature Component
Create `frontend/src/components/admin/EditKeyModal.tsx`.
- **Purpose**: Encapsulate the form logic for updating a key.
- **State**: Manages local form state (note, rate limit, models) seeded from the passed `apiKey`.
- **Composition**: Uses `ModelSelector` in its full "tags display" mode (not compact), allowing for easy management of many models.

### 3. KeyList Refactor
Simplify `frontend/src/components/admin/KeyList.tsx`.
- **State**: Replace per-row `isEditing` with a single parent `editingKey: ApiKey | null`.
- **UI**: Render the `EditKeyModal` conditionally when `editingKey` is present.
- **Actions**: The "Edit" button on a row simply sets `setEditingKey(key)`.

## Implementation Steps
1.  **Create `Modal` UI component**: Build the base dialog infrastructure.
2.  **Create `EditKeyModal`**: specific business logic wrapper around the UI modal.
3.  **Refactor `KeyList`**: Remove inline edit code and integrate the new modal flow.
4.  **Verify**: Ensure model selector works correctly inside the new modal (z-index check).
