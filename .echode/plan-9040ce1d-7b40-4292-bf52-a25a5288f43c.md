# Frontend UI Redesign & Model Selector Refactor

## Overview
Redesign the Admin Dashboard UI to improve spacing, layout, and usability. The primary focus is refactoring the `ModelSelector` component from a cramped, all-in-one input to a modular, spacious "Multi-Select" pattern. This will decouple the selection logic from the display logic, solving the "compact and tight" issue.

## Goals
1.  **Fix Model Selector**: Transform the compact dropdown into a robust Multi-Select component with search, clear separation of trigger and selected values, and better spacing.
2.  **Improve Dashboard UI**: Increase layout density, add responsive padding, and ensure a modern "flat" aesthetic using the `zinc` color scale.
3.  **Modular Architecture**: Split the monolithic `ModelSelector.tsx` into focused sub-components.

## Architecture & File Structure

### 1. New Modular Model Selector
Refactor `frontend/src/components/admin/ModelSelector.tsx` into a directory structure:
```
frontend/src/components/admin/model-selector/
├── index.ts              # Public API
├── model-selector.tsx    # Main container & logic
├── selector-trigger.tsx  # The clickable button (fixed height)
├── selector-dropdown.tsx # The searchable list (absolute positioned)
└── tags-display.tsx      # The area showing selected chips (below input)
```

### 2. Component Updates
*   **`CreateKeyForm.tsx`**: Update layout to place `TagsDisplay` *below* the selector trigger for better visibility and management.
*   **`KeyList.tsx`**: Update the inline edit mode to use the new `ModelSelector` in a "summary only" mode (showing "X selected" text instead of expanding rows).
*   **`Dashboard.tsx`**: Increase container `max-width` and vertical padding for a more "breathable" design.

## Implementation Steps

### Step 1: Create Model Selector Sub-components
*   **`selector-trigger.tsx`**: A clean button component that displays placeholder text or a summary (e.g., "3 models selected").
*   **`selector-dropdown.tsx`**: An absolute-positioned div containing:
    *   A search input (`Input`) at the top.
    *   A scrollable list of models with checkboxes.
    *   "Select All" / "Clear" actions.
*   **`tags-display.tsx`**: A flex container rendering `Badge` components with "X" buttons.

### Step 2: Implement Main Model Selector Logic
*   **`model-selector.tsx`**:
    *   Manage open/close state.
    *   Manage search term state.
    *   Compose the Trigger and Dropdown.
    *   *Note*: It will NOT render tags inside the trigger anymore.

### Step 3: Update `CreateKeyForm`
*   Replace the old `ModelSelector` import.
*   Layout:
    ```tsx
    <Label>Allowed Models</Label>
    <ModelSelector ... /> {/* Trigger & Dropdown only */}
    <TagsDisplay ... />  {/* Full width display below */}
    ```

### Step 4: Update `KeyList`
*   Replace the old `ModelSelector` import.
*   For the "Edit" mode in the table, use `ModelSelector` but *do not* render `TagsDisplay` (or render a tooltip version) to keep the table row compact.

### Step 5: Dashboard Visual Polish
*   Update `frontend/src/components/admin/Dashboard.tsx`.
*   Change `p-8` to `py-12 sm:py-16`.
*   Change `max-w-5xl` to `max-w-6xl`.
*   Ensure consistent `zinc` colors and rounded corners.
